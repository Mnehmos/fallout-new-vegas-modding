# Dense Combat Perk Redesign

## Purpose

This pass assumes a second mod will increase enemy counts across the Mojave.
Because of that, perks should be slightly OP by vanilla standards.
The player needs stronger kill chains, stronger sustain, better ammo economy, and better crowd control to survive denser encounters without turning every fight into a slow attrition check.

## Design Rules

- Every perk should improve at least one of: kill speed, action economy, sustain, mobility, crowd control, or ammo economy.
- Pure utility perks still need combat relevance somewhere in the loop.
- Requirements can be lowered if the perk arrives too late for the build that wants it.
- "Slightly OP" means the perk should feel unfair for the player, but fair inside a world with 2x-3x the normal combat pressure.

## Implementation Priorities

1. Start with requirement changes and simple numeric buffs in xEdit.
2. Then add kill-chain, AP refund, and sustain mechanics to anchor perks.
3. Save the more theatrical scripting work for morale, fear, chain reactions, and temporary buff windows.
